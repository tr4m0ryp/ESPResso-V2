OUTPUT FORMAT

Respond with ONLY a JSON array of transport leg objects. No text before
or after the JSON. Each leg object has exactly these 13 fields:

{
  "leg_index": int,          sequential starting at 0
  "material": str,           EXACT material name from input (every material must appear)
  "from_step": str,          processing step at origin
  "to_step": str,            processing step at destination
  "from_location": str,      "City, Country"
  "to_location": str,        "City, Country"
  "from_lat": float,         WGS84 latitude of origin
  "from_lon": float,         WGS84 longitude of origin
  "to_lat": float,           WGS84 latitude of destination
  "to_lon": float,           WGS84 longitude of destination
  "distance_km": float,      total distance for this leg
  "transport_modes": [str],  ordered modes used
  "reasoning": str           narrative describing the journey
}

- Material names in "material" field are EXACT copies from the input

  in Bangladesh)
- Polyester chain: 2 legs (extrusion/texturing in Taiwan to assembly
  in Bangladesh)
- Both materials converge at garment_assembly in Dhaka
- Final warehouse leg carries assembled product to Rotterdam (EU)
- All chains end at the SAME destination (Rotterdam, Netherlands)
- Multi-modal legs show segment-by-segment reasoning with distances
- leg_index is sequential across all legs (0-5)
- Distances reflect actual transport routes, not straight-line distance
